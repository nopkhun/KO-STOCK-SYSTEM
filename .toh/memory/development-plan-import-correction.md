# Plan: กรณีนำเข้าสินค้าผิด — วิธีปรับยอดหรือนำสินค้าออก

**วันที่:** 2026-02-02  
**คำถาม:** ในกรณีที่นำเข้าสินค้าผิด ควรใช้วิธีไหนในการปรับยอดหรือนำสินค้าออก

---

## สรุปคำแนะนำ

ระบบมี **สองช่องทาง** ที่ใช้ได้ทั้งคู่ ขึ้นกับว่าต้องการ “ประวัติแบบไหน” และ “ความถนัดในการกรอกข้อมูล”:

| วิธี | ใช้เมื่อไหร่ | ประวัติที่ได้ | ขั้นตอนสั้นๆ |
|------|----------------|----------------|----------------|
| **1. เบิกของ (นำออก)** | ต้องการ “นำของที่นำเข้าผิดออก” ให้ชัดเจน มีประวัติเป็น “นำออก” | ประเภท **นำออก** (out) | Dashboard/คลังสินค้า → เบิกของ → เลือกสินค้า + จำนวนที่ต้องการนำออก → ยืนยัน |
| **2. ตรวจนับสต็อกจริง (ปรับยอด)** | ต้องการ “ให้ยอดในระบบตรงกับของจริงที่หน้าร้าน” | ประเภท **ปรับยอด** (adjust) | แท็บ ตรวจนับสต็อกจริง → กรอกจำนวนจริงที่ถูกต้องของแต่ละรายการ → บันทึกและปรับยอด |

---

## วิธีที่ 1: เบิกของ (นำออก) — แนะนำเมื่อ “นำเข้าผิดแล้วอยากเอาออก”

### เหมาะกับ
- นำเข้าผิดทั้งหมด → ต้องการนำออกเท่าที่นำเข้า
- นำเข้าผิดบางส่วน (เช่น นำเข้า 100 แต่ควรเป็น 50) → เบิกออก 50
- ต้องการให้ประวัติชัดว่าเป็น **“นำออก”** (เบิก/คืนของ/ยกเลิกการนำเข้า)

### วิธีทำ
1. ไปที่ **Dashboard** หรือ **คลังสินค้า (FIFO)**
2. กด **เบิกของ** (หรือปุ่มเบิกออกที่รายการสินค้า)
3. เลือก **สินค้า** ที่นำเข้าผิด
4. กรอก **จำนวนที่ต้องการนำออก** (หน่วยตามหน่วยของสินค้าในระบบ)
5. กด **ยืนยัน**

### ระบบทำงานอย่างไร
- ใช้ **FIFO**: ลดจาก Lot ที่เข้ามาก่อน (เก่าที่สุด) ก่อน
- บันทึกประวัติเป็นประเภท **นำออก (out)**
- สต็อกลดตามจำนวนที่เบิก

---

## วิธีที่ 2: ตรวจนับสต็อกจริง (ปรับยอด) — แนะนำเมื่อ “อยากให้ยอดตรงของจริง”

### เหมาะกับ
- รู้ **จำนวนจริงที่หน้าร้าน** (นับได้แล้ว)
- ต้องการให้ระบบ **ปรับยอดให้ตรงกับของจริง** โดยไม่สนว่าเหตุผลคือนำเข้าผิด หรือของหาย ฯลฯ

### วิธีทำ
1. ไปที่แท็บ **ตรวจนับสต็อกจริง**
2. กรอก **จำนวนจริง** ที่ถูกต้องของแต่ละสินค้า (ถ้านำเข้าผิดทั้งหมด อาจกรอก 0 หรือจำนวนที่ถูกต้อง)
3. กด **บันทึกและปรับยอด**

### ระบบทำงานอย่างไร
- เปรียบเทียบ **จำนวนในระบบ** กับ **จำนวนจริงที่กรอก**
- **ผลต่างเป็นลบ** (ของจริงน้อยกว่า) → ระบบใช้ FIFO ลด Lot ตามผลต่าง
- **ผลต่างเป็นบวก** (ของจริงมากกว่า) → ระบบเพิ่ม Lot “ปรับยอดจากการนับจริง”
- บันทึกประวัติเป็นประเภท **ปรับยอด (adjust)** พร้อมหมายเหตุ เช่น “เดิม X -> Y”

---

## เปรียบเทียบและข้อควรระวัง

| หัวข้อ | เบิกของ (นำออก) | ตรวจนับสต็อกจริง (ปรับยอด) |
|--------|-------------------|-----------------------------|
| **ประวัติ** | เห็นเป็น “นำออก” ชัดเจน | เห็นเป็น “ปรับยอด” |
| **การกรอก** | กรอก “จำนวนที่ต้องการนำออก” | กรอก “จำนวนจริงที่หน้าร้าน” (ทั้งรายการ) |
| **จำนวนรายการ** | ทำทีละสินค้า (เบิกทีละรายการ) | กรอกได้หลายรายการพร้อมกัน แล้วบันทึกครั้งเดียว |
| **ความหมายทางบัญชี** | “เราตัดสินใจนำของออก X หน่วย” | “เรานับได้จริง Y หน่วย จึงปรับระบบให้เป็น Y” |

### ข้อควรระวัง
- **เบิกของ:** ต้องกรอกจำนวนไม่เกินสต็อกคงเหลือ ระบบจะแจ้ง “สต็อกไม่เพียงพอ” ถ้าเบิกเกิน
- **ตรวจนับสต็อกจริง:** ถ้ากรอกจำนวนจริงผิดอีกครั้ง จะทำให้ยอดผิดซ้ำ ควรนับของจริงให้ตรงก่อนบันทึก

---

## สรุปการ chọn ใช้

- **ต้องการ “ลบของที่นำเข้าผิดออก” และให้ประวัติเป็น “นำออก”**  
  → ใช้ **เบิกของ**
- **ต้องการ “ให้ยอดในระบบตรงกับของที่หน้าร้าน” (รวมกรณีนำเข้าผิด)**  
  → ใช้ **ตรวจนับสต็อกจริง**

ทั้งสองวิธีใช้กลไก FIFO และการอัปเดต Inventory/Transactions เดียวกับที่ระบบใช้อยู่แล้ว ไม่ต้องพัฒนาฟีเจอร์ใหม่สำหรับกรณี “นำเข้าผิด” โดยเฉพาะ ถ้าต้องการให้มีปุ่มหรือข้อความช่วยแนะนำใน UI (เช่น คำแนะนำสั้นๆ ในหน้า Dashboard หรือใน modal นำเข้า) สามารถทำเป็น Phase แยกในแผนพัฒนาต่อไปได้
